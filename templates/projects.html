Google Maps Query

Um projeto / Ferramenta de linha de comando de Web scraping que extrai contatos e informações de negócios do Google Maps com base em localização e nicho. O projeto possui 2 versões (Java / Python), ambas usando Selenium com o código base quase idêntico, com diferenças na interface.

Esse projeto foi feito com as seguintes tecnologias:

Python - Selenium, Tkinter, Pandas;
Java - Selenium, FuzzyScore, XSSFWorkbook.

A versão em python é a original é bem básica, tendo como diferencial incluso a ferramenta Business Client Capture, que usa os arquivos gerados para enviar mensagem no Whatsapp para os negócios, com possibilidade de mensagens personalizadas e mensagens programadas.
A versão Java foi uma adaptação feita para suprir a necessidade de um projeto de faculdade, ele não tem incluso a ferramenta que chama os contatos, porém ele é muito mais amigável em questão que, a exigência da faculdade é a documentação, e uma seção FAQ (perguntas frequentes) onde eu implementei o esquema "FuzzyScore" para que as perguntas não tenham que ser escritas exatamente para ser identificadas. Caso queira testar elas, baixe o arquivo .exe, ou use o código condizente disponível no github.

Plataforma Dinâmica de Gestão de Equipes - IBM

Um projeto de visualização e gestão da alocação de equipes técnicas, com foco em sua distribuição operacional. A ferramenta tem uma peculiaridade devido às necessidades da empresa, a interface é dinâmica, com adição flexível de colunas e linhas, adaptando-se a estruturas organizacionais voláteis.

Stack:

Python, Flask, Podman, IBM DB2

Essa nescessidade volatil de colunas e linhas foi um desafio interessante de resolver, e nós tinhamos que utilizar o DB2 como banco de dados, o mesmo é relacional, mas nós usamos um objeto CLOB (Character Large Object) para armazenar os dados, e o uso de Flask para criar uma API que interage com o banco de dados DB2, guardando os dados de forma dinâmica, e retornando eles para o frontend, que é feito em HTML puro, com CSS e JavaScript.

A applicação foi transformada em um container Podman, para facilitar o deploy e a escalabilidade da mesma, utilizando o Podman devido a exigencias da empresa. A nescessidade de usar container se leva ao fato que nós iriamos hospedar a aplicação em um servidor cloud IBM Code Engine.

Essas dificuldades e desafios exigiram criatividade, principalmente na hospedagem, gerenciamento do banco de daods e na hora da connexão com o front-end da applicação e formação das funções da Api, porem ela permite de forma amigalvel que um gerente altere quando nescessario a estrutura da equipe.

TLS SLA Scripts - IBM

Um projeto de automação/pipelines de informação de scripts para a equipe de TLS da IBM, com o objetivo de automatizar o processo de criação e atualização de SLAs (Service Level Agreements) para clientes. São 3 scripts que extraem informações de um arquivo Excel ou CSV e as organizam em arquivos separados por cliente e tipo de análise, facilitando a gestão e o acompanhamento dos SLAs.

Stack:

Python, RapidFuzz, Numpy, Pandas

Pipeline de informações com Python, Extração e análise de dados, uma ferramenta de linha de comando CLI Que distrubui para 3 scripts diferentes, sendo eles FIN, SF4 e SLA, cada um com uma função específica e analises especiaes, um arquivo originario e pasta (uma para cada analise, cada uma com um arquivo excel pra cliente).

Foi um trabalho de melhoria de um script antigo que ja existia, tive que melhorar a performance dele, e a usabilidade, adicionar novas funções e etc. Um dos maiores desafios foi entender as regras de negocio e como isso se aplicava ao script.
